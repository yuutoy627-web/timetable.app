# 次のステップ

Google認証の設定が完了しました！次は以下の手順を進めてください。

## Step 1: 動作確認

1. **開発サーバーが起動しているか確認**
   - ターミナルで `npm run dev` が実行されているか確認
   - 起動していない場合は起動してください

2. **ブラウザでアプリケーションにアクセス**
   - `http://localhost:3000` を開く

3. **Google認証をテスト**
   - 「Googleでログイン」ボタンをクリック
   - Googleアカウントの選択画面が表示されることを確認
   - アカウントを選択してログイン

4. **ログイン後の画面確認**
   - ダッシュボードにリダイレクトされることを確認
   - ヘッダーにユーザー情報が表示されることを確認

## Step 2: データベーススキーマの設定

タイムテーブルを保存するために、データベーススキーマを設定する必要があります。

### 手順

1. **Supabaseダッシュボードにアクセス**
   - [https://supabase.com/dashboard](https://supabase.com/dashboard)
   - プロジェクトを開く

2. **SQL Editorを開く**
   - 左サイドバーで「SQL Editor」をクリック
   - または「Database」→「SQL Editor」

3. **スキーマを実行**
   - 「New query」をクリック
   - `docs/schema.sql` ファイルの内容をコピー
   - SQL Editorに貼り付ける
   - 「Run」ボタンをクリック（または `Ctrl + Enter`）

4. **実行結果を確認**
   - 「Success. No rows returned」と表示されれば成功
   - エラーが表示された場合は、エラーメッセージを確認

## Step 3: タイムテーブル作成のテスト

1. **新規タイムテーブルを作成**
   - ダッシュボードで「新規作成」ボタンをクリック
   - または `http://localhost:3000/timelines/new` にアクセス

2. **ウィザードで入力**
   - Step 1: ジャンルを選択（例：音響(PA)）
   - Step 2: 基本情報を入力
   - Step 3: イベントを追加（任意）
   - Step 4: 確認して保存

3. **保存結果を確認**
   - ダッシュボードに戻り、作成したタイムテーブルが表示されることを確認

## Step 4: 共有機能のテスト

1. **タイムテーブルの詳細を開く**
   - ダッシュボードで作成したタイムテーブルをクリック

2. **共有機能をテスト**
   - 右上の「共有」ボタンをクリック
   - 「公開する」をクリック
   - リンクをコピー

3. **共有リンクでアクセス**
   - 別のブラウザ（またはシークレットモード）でリンクを開く
   - ログインなしでタイムテーブルが表示されることを確認

## トラブルシューティング

### エラー: "relation does not exist"

**原因**: データベーススキーマが設定されていない

**解決方法**:
- `docs/schema.sql` をSupabaseのSQL Editorで実行してください

### エラー: "permission denied"

**原因**: Row Level Security (RLS) ポリシーが正しく設定されていない

**解決方法**:
- `docs/schema.sql` を完全に実行してください（RLSポリシーも含まれています）

### タイムテーブルが保存されない

**原因**: データベーススキーマが設定されていない、または認証エラー

**解決方法**:
1. データベーススキーマが設定されているか確認
2. ブラウザの開発者ツール（F12）でエラーを確認
3. ターミナルのエラーメッセージを確認
