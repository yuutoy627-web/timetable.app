# クイックスタートガイド

## 実際にアプリケーションを使うための手順

### 1. 開発サーバーの起動

ターミナルで以下のコマンドを実行：

```bash
npm run dev
```

**期待される出力**:
```
  ▲ Next.js 14.x.x
  - Local:        http://localhost:3000
  - Ready in X.XXs
```

### 2. ブラウザでアクセス

以下のURLをブラウザで開きます：

**開発環境のURL:**
```
http://localhost:3000
```

### 3. 主要なページとリンク

#### ホームページ
```
http://localhost:3000/
```
- アプリケーションのトップページ
- ログインしていない場合は「Googleでログイン」ボタンが表示されます

#### ダッシュボード（ログイン後）
```
http://localhost:3000/dashboard
```
- 作成したタイムテーブルをジャンル別に一覧表示
- 各タイムテーブルの共有リンクをコピーできます

#### 新規タイムテーブル作成
```
http://localhost:3000/timelines/new
```
- ウィザード形式でタイムテーブルを作成
- Step 1: ジャンル選択
- Step 2: 基本情報入力
- Step 3: イベント追加
- Step 4: 確認と保存

#### タイムテーブル詳細（共有リンク）
```
http://localhost:3000/timelines/[タイムテーブルID]
```
例：
```
http://localhost:3000/timelines/123e4567-e89b-12d3-a456-426614174000
```
- タイムテーブルの詳細を表示
- 公開されている場合はログインなしで閲覧可能
- 作成者は「共有」ボタンからリンクをコピー・公開設定が可能

### 4. 実際の使い方

#### タイムテーブルを作成する

1. **ログイン**
   - `http://localhost:3000` にアクセス
   - 「Googleでログイン」をクリック
   - Googleアカウントで認証

2. **新規作成**
   - ヘッダーの「新規作成」をクリック
   - または `http://localhost:3000/timelines/new` に直接アクセス

3. **ウィザードで入力**
   - Step 1: ジャンルを選択（音響、会議、旅行など）
   - Step 2: 基本情報を入力（ジャンルに応じて項目が変わります）
   - Step 3: イベントを追加（時間枠とアクション）
   - Step 4: 内容を確認して保存

4. **保存完了**
   - 自動的にダッシュボードにリダイレクトされます

#### タイムテーブルを共有する

1. **ダッシュボードから**
   - `http://localhost:3000/dashboard` にアクセス
   - 共有したいタイムテーブルカードの「共有」ボタン（📤）をクリック
   - リンクが自動的にコピーされます

2. **詳細ページから**
   - タイムテーブルの詳細ページを開く
   - 右上の「共有」ボタンをクリック
   - 共有ダイアログで：
     - リンクをコピー
     - 「公開する」をクリックして公開設定に変更

3. **リンクを共有**
   - コピーしたリンクを他の人に送る
   - 例：`http://localhost:3000/timelines/123e4567-e89b-12d3-a456-426614174000`

#### 共有リンクでアクセスする

1. **リンクを開く**
   - 共有されたリンクをブラウザで開く
   - ログイン不要で閲覧可能（公開されている場合）

2. **閲覧**
   - タイムテーブルの詳細が表示されます
   - タイムライン（イベント）と持ち物リストを確認できます

### 5. トラブルシューティング

#### 開発サーバーが起動しない

```bash
# ポート3000が使用中の場合
lsof -ti:3000 | xargs kill -9

# 別のポートで起動
npm run dev -- -p 3001
```

#### ページが表示されない

- ターミナルでエラーメッセージを確認
- ブラウザの開発者ツール（F12）でコンソールエラーを確認
- `.env.local`ファイルが正しく設定されているか確認

#### 認証エラー

- Supabaseの設定を確認
- Google認証が有効になっているか確認
- リダイレクトURLが正しく設定されているか確認

### 6. 本番環境へのデプロイ

本番環境にデプロイする場合：

#### Vercel（推奨）

1. GitHubにリポジトリをプッシュ
2. [Vercel](https://vercel.com/)でプロジェクトをインポート
3. 環境変数を設定：
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `NEXT_PUBLIC_SITE_URL`（デプロイ後のURL）

#### 本番環境のURL例

```
https://your-app-name.vercel.app
```

デプロイ後は、このURLが実際の作業用リンクになります。

## まとめ

**開発環境:**
- ローカル: `http://localhost:3000`
- 起動コマンド: `npm run dev`

**主要なページ:**
- ホーム: `/`
- ダッシュボード: `/dashboard`
- 新規作成: `/timelines/new`
- タイムテーブル詳細: `/timelines/[id]`

**共有リンク:**
- 形式: `http://localhost:3000/timelines/[タイムテーブルID]`
- 公開設定が必要（「共有」ボタンから設定）
